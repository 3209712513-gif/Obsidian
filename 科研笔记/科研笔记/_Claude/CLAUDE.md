## /lit — 文献精读

**用途：** 精读一篇学术论文，逐词逐句讲透，匹配大一水平

**用法：** `/lit <论文标题/DOI/链接>` 或 `/lit` 后粘贴 PDF/内容

**输出：** 在 `文献/论文笔记/` 下创建精读笔记，格式如下：

### 📄 笔记结构

#### 1. 论文档案
- 标题、作者、年份、会议/期刊
- **一句话总结**（这个论文到底做了什么）

#### 2. 术语表（Glossary）
每一个你可能不懂的术语，用 `**术语** → 大白话解释` 格式列出。例如：
- **Backbone** → 特征提取的主干网络，相当于流水线的第一段
- **Ablation Study** → 控制变量法，拆掉一个部件看效果
- **End-to-End** → 输入直接出输出，中间不用人插手

#### 3. 概念深讲
对于论文涉及的每个核心概念：
- **这是什么**（一句话定义）
- **为什么需要它**（没有它会怎样）
- **直观理解**（用生活中的类比）
- **数学公式**（拆成一行一行讲，不跳步）
- **代码实现**（最小可运行示例）
- **在 CV 领域的典型应用**

#### 4. 公式推导
每一条公式：
- **符号解释**：每个字母代表什么
- **物理意义**：这个公式在算什么
- **推导过程**：从哪儿来到哪儿去，不跳步
- **直观理解**：用数值例子跑一遍
- **代码验证**：用 Python 算一遍验证

#### 5. 学习方法资源
**视频教程**（B站/YouTube）：
- 标题 + 链接 + 为什么推荐

**值得关注的老师/博主**：
- 名字 + 平台 + 主要内容方向

**推荐博客/文章**：
- 标题 + 链接 + 难度评级（⭐）

**推荐教材章节**：
- 书名 + 章节 + 页码

#### 6. 与吃水线检测的关联
- 这个方法能不能用在吃水线检测上？
- 如果能，怎么用？（思路 + 伪代码）
- 如果不能，为什么？以后有没有可能？

#### 7. 疑问区（自动记入知识库）
- 这篇论文里还有什么没懂的？
- 哪些前置知识需要补？
- 自动追加到 `_个人知识库.md` 的「待学习」清单

---

### 📚 个人知识库体系（自动维护）

| 文件 | 位置 | 作用 |
|------|------|------|
| `_个人知识库.md` | `文献/` | 记录已掌握和待学的知识点图谱 |
| `_不会清单.md` | `文献/` | 每次读文献标记的不懂之处，排序优先级 |
| `_学习资源库.md` | `文献/` | 积累的学习资源，带评分和心得 |

每次调用 `/lit` 后，我会自动：
1. ✅ 分析论文并写入精读笔记
2. ✅ 检测新术语/概念，更新「知识库」
3. ✅ 把你不懂的问题追加到「不会清单」
4. ✅ 如果有相关资源，更新「学习资源库」
5. ✅ 定期检查「不会清单」中的问题是否已解决

## /concept — 概念学习

**用途：** 讲解一个 CV/深度学习/数学概念，用大白话 + 代码示例

**用法：** `/concept <概念名>`

**输出：** 在 `概念笔记/` 下创建笔记，包含：
- 一句话解释（给完全不懂的人听）
- 为什么需要它（学习动机）
- 数学原理（轻量级，公式 + 直观解释）
- OpenCV/Python 代码示例
- 在吃水线检测中的应用场景
- 类比/记忆口诀

## /experiment — 实验记录

**用途：** 结构化记录一次实验，方便后续对比和复盘

**用法：** `/experiment`（然后描述实验内容）

**输出：** 在 `实验记录/` 下创建笔记，包含：
- 实验日期与目的
- 实验环境（操作系统、Python版本、依赖库版本）
- 数据集描述
- 方法/参数配置
- 运行结果（指标、图片路径）
- 与之前实验的对比分析
- 结论与下一步计划

## /debug — 代码调试

**用途：** 帮你分析和修复吃水线检测相关的代码问题

**用法：** `/debug <贴出报错或问题描述>`

**行为：**
1. 分析报错信息/异常表现
2. 定位问题根因
3. 给出修复方案 + 代码
4. 解释为什么会出这个问题（避免下次再犯）

## /plan — 学习计划

**用途：** 制定/调整学习路线

**用法：** `/plan` 查看当前计划；`/plan 本周我想学...` 定制新计划

**输出：** 在 `学习计划/` 下创建计划表，包含：
- 本周核心目标
- 每日任务安排（具体到看什么、写什么）
- 预期产出
- 参考资源链接
- 复习提醒

## /review — 复习回顾

**用途：** 根据 vault 中的笔记生成复习摘要和自测题

**用法：** `/review`（最近一周）；`/review <主题名>`（指定主题）

**输出：**
- 关键知识点摘要
- 3~5 道自测问答题
- 建议重点复习的薄弱环节

## /digest — 日报/周报

**用途：** 根据 vault 中的实验记录和笔记，自动生成学习日报或周报

## /img — 看图分析

**用途：** 给你看一张图片，uf 帮你分析

**用法：** `/img <图片路径>`

**支持的分析类型：**

| 你想干嘛 | 就说 |
|---------|------|
| 看检测效果 | `/img D:\实验图.jpg` → uf 评价吃水线检测得怎么样 |
| 看报错截图 | `/img D:\报错.png` → uf 帮你读报错信息 |
| 看论文图表 | `/img D:\fig2.png` → uf 解释图表含义 |
| 看中间结果 | `/img D:\mask.jpg` → uf 分析分割效果 |
| 随便分享 | `/img D:\照片.jpg` → uf 陪你聊 |

**输出：**
1. 描述我看到了什么
2. 结合你的吃水线项目给出分析/建议
3. 如果发现异常，指出可能的原因

> **注意：** 图片路径必须是本地绝对路径，如 `D:\实验\result.jpg`
