# 反直觉的时域合取：用 AND 去静止噪声

> *（此处放论文公式截图）*

## 🔍 来源

**文献：** [[文献/Tsujii2016_AutomaticDraftReading/精读笔记|Tsujii 2016 - Automatic Draft Reading Based on Image Processing]]
**位置：** Section 3.3 — Waterline Extraction（本文方法而非传统方法，但只是完整流水线中的一个环节）
**论文真实创新点：** 完整的 5 步自动化吃水线检测流程（Top-hat 吃水标志分割 + 局部 Otsu + Canny 水线提取 + LMedS 鲁棒拟合 + 模拟验船师）

## 🧠 核心思路

$$P(x,y,t) = \bigwedge_{1 \leq i \leq n} I_{dilation}[x - (a_{t-i} - a_0), y - (b_{t-i} - b_0), t-i]$$

- 对相邻多帧做**对齐**（补偿船体晃动）→ 再做**逐像素 AND 合取**
- **静止边缘**（船体划痕、焊缝）：每帧同位置出现 → AND 结果为 1 → **保留**
- **运动边缘**（水线、波纹）：每帧位置不同 → AND 结果为 0 → **滤掉**
- 最后从原始边缘图中减掉这些静止噪声 → 水线浮现

## ✨ 为什么巧妙

| 常规思路 | 本文思路 |
|---------|---------|
| 帧差法找**运动的**（水线） | 合取法找**不动的**（噪声） |
| 会同时检测到晃动噪声 | 利用多帧一致性剔除噪声 |
| 两帧相减，工程直觉 | 多帧 AND，反直觉的数学视角 |

**核心反转：** 大部分人想"把水线从背景中分割出来"，本文想的是"先把确定不动的噪声全找出来删掉，剩下的自然就是水线"——跟雕塑家"去掉多余的部分，剩下的就是作品"一个道理。

## 🔧 在本项目中的迁移可能

- 吃水线检测中，水线确实随时间波动，船体上的划痕/标记是静止的 → **完美适配**
- 当前你的方法（彭将辉）是单帧检测，可以尝试加入多帧时域处理
- 对齐步骤 $(a_t, b_t)$ 可以用模板匹配或特征点跟踪实现
- 也可以简化：不做精确对齐，直接用滑动窗口局部 AND

## 📝 补充笔记

- 对齐精度很关键：如果对齐不准，静止边缘也会被 AND 滤掉
- 本文用了吃水标志的位置来做对齐（用模板匹配找到吃水标志，以其位置为参考）
- n 的取值：文中用的是 n=3（前后各 3 帧），窗口太大则对齐误差累积，太小则去噪不充分
- 最终保留公式 $O = I \cap (P \cup F)$：当前帧边缘 ∩（过去静止 ∪ 未来静止）→ 得到噪声掩膜 → 原边缘图减去 O 即得水线
