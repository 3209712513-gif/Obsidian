# 船舶吃水线检测实践

## 一、方法流程（最终稳定版）

中值滤波去噪 -> HSV颜色分割提取红色船体 -> 形态学闭运算填补空洞 -> 连通域过滤去噪声块 -> 逐列找最低点 -> 中位数平滑 -> 画出吃水线

## 二、关键步骤详解

### 1. HSV 颜色分割
OpenCV 默认读取为 BGR 色彩空间，需转换成 HSV：

```python
hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)
```

红色船体分割条件示例：
- H 在 0~10 之间（红色色调低段）
- S 在 80~255 之间（排除灰色）
- V 在 80~255 之间（排除过暗区域）

### 2. 逐列扫描找吃水线
```python
waterline_points = []
for col in range(w_mask):
    column = red_mask[:, col]
    white_pixels = np.where(column == 255)[0]
    if len(white_pixels) > 0:
        lowest = white_pixels[-1]
        waterline_points.append((col, lowest))
```

### 3. 中位数平滑
逐列独立计算时，噪声点会直接跳到很低的位置。解决方法是对 y 坐标做中位数平滑：

```python
smoothed_points = []
window = 30
for i in range(len(waterline_points)):
    left = max(0, i - window)
    right = min(len(y_coords), i + window)
    median_y = int(np.median(y_coords[left:right]))
    smoothed_points.append((waterline_points[i][0], median_y))
```

中位数不受少数跳变噪声值的影响，比平均数更稳健。

## 三、遇到的问题与解决

### 问题 1：水面干扰
右边水面有绿藻反光颜色偏暖，H 值混入红色范围；中间白色反光模糊了船体和水面边界。

**解决：** 加强 HSV 阈值过滤 + 形态学闭运算填补船体空洞。

### 问题 2：逐列噪声跳变
每列独立计算最低点，个别列因噪声跳到很低的位置，吃水线不平滑。

**解决：** 中位数平滑（window=30）。

## 四、Canny 方法的尝试与反思

用 Canny 边缘检测 -> 找白色像素最多的行（吃水线是图里最长的一条横线，理论上白色像素最多），但水面波纹、藻类边缘加起来比吃水线还多，这个假设在实际场景中失效。

最终采用**标准差突变法**（自行推导），逻辑更清晰，更符合吃水线的物理特征。
