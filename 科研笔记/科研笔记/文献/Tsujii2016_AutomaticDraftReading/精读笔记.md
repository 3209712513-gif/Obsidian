# Automatic draft reading based on image processing

**作者：** Takahiro Tsujii, Hiromi Yoshida, Youji Iiguni（大阪大学）
**年份：** 2016
**来源：** Optical Engineering, Vol. 55(10), 104104
**DOI：** [10.1117/1.OE.55.10.104104](https://doi.org/10.1117/1.OE.55.10.104104)

---

## 📝 一句话总结

这篇论文提出了一套**完整的自动吃水线读取方法**，用**形态学运算（Top-hat 变换）**检测吃水标志，用 **Canny 边缘检测 + LMedS 鲁棒估计**找水线，精度 <1cm，并且模拟了验船师的读数过程，让船东和收货人都能信服。

## 🧠 核心概念深讲

### 1️⃣ Top-hat 变换（核心方法）

**是什么：**
Top-hat 变换是一种形态学运算，专门用来检测图像中**比结构元素小的亮或暗的物体**。吃水标志是船体上的细长白色/黑色条纹，正好符合这个条件。

**为什么需要它：**
传统方法（Otsu 全局二值化）的问题是：整张图里除了吃水标志和船体，还有水面、阴影、倒影等其他区域，直方图不是双峰的，Otsu 会分错。Top-hat 变换只看局部，不受远处水面干扰。

**数学定义：**

**白 Top-hat（检测亮色吃水标志）：**
$$T_{white}(x,y) = I(x,y) - I_{opening}(x,y)$$

- $I(x,y)$ = 原始图像
- $I_{opening}(x,y)$ = 先腐蚀再膨胀（开运算）的结果
- 开运算去掉了小而亮的物体，原图减去它 = 只保留小而亮的物体

**黑 Top-hat（检测暗色吃水标志）：**
$$T_{black}(x,y) = I_{closing}(x,y) - I(x,y)$$

- $I_{closing}(x,y)$ = 先膨胀再腐蚀（闭运算）的结果
- 闭运算填充了小而暗的区域，它减去原图 = 只保留小而暗的物体

**直观类比：**
想象一张白纸上有些小黑点。你用一张砂纸（结构元素）在整个纸上磨一下（开运算），小黑点被磨掉了。然后用原纸减去磨过的纸——剩下的就是那些小黑点。

**结构元素：**
论文用的是**圆形结构元素**，半径 $p = Y/30$（$Y$ 是图像高度像素数）。为什么这么设？因为 30cm 高度至少包含两个吃水标志，$Y/30$ ≈ 1cm，比吃水标志的笔画宽度（2cm）大，能覆盖住标志。

```python
import cv2
import numpy as np

# 圆形结构元素
radius = 30  # 根据图像尺寸调整
kernel = cv2.getStructuringElement(cv2.MORPH_ELLIPSE, (radius*2+1, radius*2+1))

# 白 Top-hat
opening = cv2.morphologyEx(img, cv2.MORPH_OPEN, kernel)
white_tophat = cv2.subtract(img, opening)

# 黑 Top-hat
closing = cv2.morphologyEx(img, cv2.MORPH_CLOSE, kernel)
black_tophat = cv2.subtract(closing, img)
```

### 2️⃣ LMedS（Least Median of Squares）鲁棒估计

**是什么：**
一种直线拟合方法。最小二乘法（Least Squares）拟合直线时，**一个离群点就能把直线拉偏**。LMedS 不怕离群点——它能自动忽略那些"瞎跳"的噪点。

**为什么需要它：**
水线提取后，除了真正的水线像素外，还有海面波纹、反光等产生的噪点边缘。最小二乘法会被这些噪点带偏，LMedS 不会。

**数学原理：**

最小二乘法：找到一条直线，使**所有点到直线的距离平方和**最小
$$\min \sum_{i=1}^{n} r_i^2$$ 
（$r_i$ 是第 $i$ 个点到直线的垂直距离）

LMedS：找到一条直线，使**所有点到直线距离的"中位数"**最小
$$\min \ \text{median}(r_1^2, r_2^2, ..., r_n^2)$$

**关键区别：**
- 最小二乘法：**"所有人都投票"**——一个坏点就能把结果带歪
- LMedS：**"投票取中位数"**——50% 的噪点都不影响结果

**直观类比：**
一群人站在一起，你要画一条线描述他们的"平均位置"。
- 最小二乘法：有一个人站得特别远，为了照顾他，整条线被拉过去了
- LMedS：不管那个站得远的人，只看大多数人的中间位置

### 3️⃣ 时域静止边缘去除（Sec 3.3 —— 流程中的一个工程技巧）

**是什么：**
水线随时间波动，而船体上的划痕、焊缝等噪声边缘是静止的。论文利用多帧信息区分它们。

**方法：**
对第 $t$ 帧，分别**向前看 $n$ 帧**和**向后看 $n$ 帧**：

$$P(x,y,t) = \bigwedge_{1 \leq i \leq n} I_{dilation}[x - (a_{t-i} - a_0), y - (b_{t-i} - b_0), t-i]$$

$$F(x,y,t) = \bigwedge_{1 \leq i \leq n} I_{dilation}[x - (a_{t+i} - a_0), y - (b_{t+i} - b_0), t+i]$$

- $P$ = 过去 $n$ 帧中静止的边缘
- $F$ = 未来 $n$ 帧中静止的边缘
- $I_{dilation}(x,y,t)$ = 第 $t$ 帧的 Canny 边缘膨胀结果
- $(a_t, b_t)$ = 第 $t$ 帧吃水标志的位置（用于对齐）

最终保留的边缘：
$$O = I \cap (P \cup F)$$

**大白话：**
> 拍了 100 帧视频。船体上的划痕每帧都在同一个位置，而水线上下波动。取所有帧的"公共部分"，划痕还在那里，水线反而被滤掉了——反其道而行之，先把静止噪点去掉，剩下的就是水线候选。

### 4️⃣ Otsu 二值化（局部版）

**是什么：**
Otsu 方法自动找一个阈值，把灰度图分成黑（背景）和白（前景）两部分。它找的阈值让"类内方差最小、类间方差最大"。

**传统的 Otsu（全局）的问题：**
整张图包含船体、水面、阴影——直方图不是双峰，Otsu 分不好。

**论文的 Otsu（局部）：**
先用 Top-hat 找到吃水标志候选区域，**只在这些局部小区域内**做 Otsu 二值化。局部区域内只有吃水标志和船体两种东西，直方图是双峰的，Otsu 效果好。

```python
# 局部 Otsu 二值化
def local_otsu(img, bounding_boxes):
    result = np.zeros_like(img)
    for box in bounding_boxes:
        x, y, w, h = box
        roi = img[y:y+h, x:x+w]
        _, binary = cv2.threshold(roi, 0, 255, cv2.THRESH_BINARY + cv2.THRESH_OTSU)
        result[y:y+h, x:x+w] = binary
    return result
```

## 📐 公式拆解

### 噪声段去除条件

论文用三个条件判断一个连通区域是不是"噪点"：

**条件 1：宽高比**
$$T_1 \leq w/h \leq T_2$$

- $w$ = 区域的宽度（像素）
- $h$ = 区域的高度（像素）
- $T_1 = 0.1$，$T_2 = 1$
- **原因：** 吃水标志"1"的宽高比最小是 0.2，所以设 $T_1=0.1$ 保留它。吃水标志的高度通常大于宽度，所以 $T_2=1$

**条件 2：面积**
$$T_3 \leq s$$

- $s$ = 区域的面积（像素数）
- $T_3 = 10$
- **原因：** 小于 10 个像素的小点肯定是噪点，直接丢掉

## 🔗 与吃水线检测的关联

**这篇论文和你做的项目高度相关！** 下面是对比分析：

### 方法对比

| 模块 | 你这篇论文（Tsujii 2016） | 你之前做的（彭将辉） |
|------|------------------------|-------------------|
| 预处理 | 中值滤波 | 中值滤波 ✅ 一致 |
| 船体检测 | Top-hat 形态学 + 局部 Otsu | HSV 颜色分割 |
| 水线检测 | Canny + 时域静止边缘去除 + LMedS | 逐列扫描 + 中位数平滑 |
| 吃水标志 | 识别数字和 M | 你的方法不涉及吃水标志 |
| 鲁棒性 | 多帧投票 + LMedS | 中位数平滑 |

### 你的项目可以借鉴的点

1. **⭐ 时域静止边缘去除**：你的方法目前是单帧检测，可以试试用多帧信息提高稳定性
2. **⭐ LMedS**：你目前用的是中位数平滑做后处理，试试能不能用 LMedS 替代逐列扫描
3. **形态学 Top-hat**：如果你的船体颜色不固定（不同船颜色不同），Top-hat 比 HSV 更通用
4. **局部 Otsu**：比全局阈值更鲁棒，可以试试

### 论文局限性
- **处理速度慢**：1800 帧要 670 秒（≈ 2.7 fps），实时性差
- **需要吃水标志**：不是所有船都有清晰的白色/黑色吃水标志
- **摄像机需固定**：提出的对齐方法假设摄像机相对船体稳定
